Hana-Bi — Prioritized Improvement Roadmap

Ordered by impact and pre-launch necessity. Scroll down to see all phases.

Phase 1 — Must Fix Before Launch (Broken)

🔴 Critical Issues

Critical
Checkout is fundamentally broken The Stripe route always charges a single hardcoded STRIPE_PRICE_ID regardless of what's in the cart. Real cart items, variants, and quantities are completely ignored. Needs dynamic line_items built from the Zustand cart.
Critical
No Stripe webhook handler Without a /api/webhooks/stripe route, there's no reliable way to confirm payment success. The current success page trusts the URL parameter alone — easily spoofed. Need to verify session_id server-side.
Backend
Cart resets on every page refresh The Zustand store is in-memory only. Users lose their cart when they reload. Add localStorage persistence via zustand/middleware persist.
Backend
Shopify checkout mutation is unused createCheckout() in shopify.ts is correctly implemented but never called. The app bypasses Shopify's native checkout entirely and goes straight to Stripe — losing inventory management and order tracking.
Phase 2 — Code Quality & Architecture

🟡 Foundational Cleanup

Code
6,500-line products.ts fallback file This is the single largest file in the project. Split into per-collection JSON files or move to a local SQLite/JSON database. Lazy-load with dynamic import() so it doesn't bloat the initial bundle.
Code
Two animation libraries doing overlapping work Framer Motion and GSAP are both installed. Pick one as the primary — Framer Motion is already the better fit for React component animations. Reserve GSAP only for scroll-based timelines if needed.
Code
Environment variable validation Currently spread across multiple files with console.warn/error. Centralize into a single validated env.ts module that throws clearly on startup if required vars are missing.
Phase 3 — Performance & SEO

🟢 Visibility & Speed

Perf
No metadata / Open Graph config Product pages and the home page have no title, description, or OG image tags. Add generateMetadata() to each route and a root metadata object in layout.tsx.
Perf
Static generation not leveraged Product pages use dynamic rendering by default. Add generateStaticParams() to /product/[slug] and /projects/[slug] for static generation with ISR revalidation (already set to 60s in shopify.ts).
Perf
No sitemap or robots.txt Add app/sitemap.ts and app/robots.ts — both are trivial in Next.js 13+ App Router and meaningfully help SEO for a new store.
Phase 4 — UX Polish

🟣 User Experience

UX
No loading/skeleton states Product pages and shop grid have no loading.tsx or skeleton components. Add these so the editorial feel isn't broken by layout shift during data fetching.
UX
No error boundaries on shop/product pages If Shopify is down and fallback data doesn't cover the slug, the user gets a blank/crashed page. Add error.tsx at the route segment level.
UX
Mobile cart / checkout flow untested The cart drawer and product size selector need audit on small screens. Drawer swipe-to-close, tap target sizes, and sticky CTA button on product pages are common pain points.
Phase 5 — New Features

🩷 Enhancements

Feature
Product search Shopify Storefront API supports searchProducts. Add a search bar + results page — especially useful once the catalog grows.
Feature
Wishlist / save for later localStorage-backed wishlist with a heart icon on product cards. Low complexity, high perceived value for a fashion brand.
Feature
Newsletter signup A minimal email capture (Mailchimp/Resend) in the footer or as a slide-in. Particularly valuable pre-launch for building a drop waitlist.
Feature
Shop filter improvements Add URL-synced filters (size, price range, collection) so filtered views are shareable and back-navigable.
